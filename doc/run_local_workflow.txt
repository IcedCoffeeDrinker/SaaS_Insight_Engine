########################################################

## SETUP ##

# Create virtual environment
python -m venv venv

# Activate it (Windows)
venv\Scripts\activate

# Activate it (Unix/MacOS)
source venv/bin/activate # for fish shell add '.fish'

# Navigate to backend directory
cd backend

# Install dependencies
pip install -r requirements.txt

## Frontend Setup ##

# Navigate to frontend directory
cd ..
cd frontend

# Install Node dependencies
npm install

# Set up environment variables
# Copy .env.example to .env in project root
cp ../.env.example ../.env
# Edit .env with your API keys
# Make sure to include:
# - Stripe Keys (Secret, Publishable, Webhook Secret)
# - API Keys (OpenAI, Reddit, DataForSEO)
# - JWT Secret Key (generate one using 'openssl rand -hex 32')
# - Mail Server details (Server, Port, TLS, Username, Password/App Password, Sender)

# Copy .env.example to .env in frontend directory
cp .env.example .env
# Add your Stripe publishable key to .env
# REACT_APP_STRIPE_PUBLISHABLE_KEY=<YOUR_STRIPE_PUBLISHABLE_KEY>
# Optionally set REACT_APP_API_URL if backend isn't on localhost:5000

# Return to root directory
cd ..

## RUN LOCALLY ##

# Terminal 1 - Start Backend (from project root)
# Ensure venv is active (source venv/bin/activate or activate.fish)
cd backend
flask run

# Terminal 2 - Start Frontend (from project root)
# Ensure venv is active (needed for node env? No, only python)
cd frontend
npm start

# Terminal 3 (Optional) - Run Stripe CLI for Webhook Testing
# Install: https://docs.stripe.com/stripe-cli
# Login: stripe login
# Forward events: stripe listen --forward-to http://localhost:5000/api/webhook

## Local vs Production Configuration ##

# The app is configured to work in both local development and production environments:
# - In development, API calls default to http://localhost:5000
# - In production, API calls use REACT_APP_API_URL from environment variables
# - CORS is configured to accept requests from both localhost:3000 and production domain

## Troubleshooting ##

# If CORS errors occur:
# - Check that backend is running on http://localhost:5000
# - Check that frontend is running on http://localhost:3000
# - Verify CORS configuration in backend/app.py (allows localhost:3000 and FRONTEND_URL)

# If Stripe doesn't work:
# - Verify .env files exist in both root and frontend directories
# - Check that Stripe keys (Secret, Publishable) are correctly set in the root .env
# - Check that REACT_APP_STRIPE_PUBLISHABLE_KEY is set in frontend/.env
# - Check that STRIPE_WEBHOOK_SECRET is set in the root .env for webhook verification.
# - If testing locally, ensure Stripe CLI is running and forwarding events correctly.

# If Authentication fails:
# - Check JWT_SECRET_KEY is set in the root .env file.
# - Check browser console for JWT-related errors.
# - Verify backend logs for authentication errors.

# If Password Reset emails aren't sent:
# - Check MAIL_* variables are correct in the root .env file.
# - Ensure your email provider (e.g., Gmail) allows SMTP access.
# - Check if 2FA requires an App Password for MAIL_PASSWORD.
# - Check backend logs for mail sending errors.

# If Tailwind styles don't appear:
# - Restart frontend development server
# - Verify tailwind.config.js and postcss.config.js exist

# If 'python' command not found:
# - Make sure Python is added to your PATH
# - Try using 'python3' instead

# If 'npm' command not found:
# - Make sure Node.js is added to your PATH
# - Try restarting your terminal

# If port 3000 is already in use:
# Windows: netstat -ano | findstr :3000
# Unix/MacOS: lsof -i :3000
# Then kill the process using that port

# If npm start fails with dependency errors:
# - Try clean install:
#   rm -rf node_modules package-lock.json
#   npm install
